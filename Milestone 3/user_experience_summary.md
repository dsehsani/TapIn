## TapIn — Milestone 3 Summary
 
### Quantitative Findings
 
Focusing on quantitative data first, our metrics for daily active users and retention indicate strong repeated engagement among users. Specifically, we noticed that the games accounted for the majority of actions. For instance, on March 11th, Wordle and Pipes games played accounted for nearly 75% of all actions on the app. While events and articles did have interactions, this signifies that they are not yet as sticky of a feature as the games. Echo game's metrics fall behind the other two games in comparison.
 
### Interview Findings
 
Moving onto our interviews, nearly every user cited the same pain point: finding out what's happening on campus is a scattered process from Instagram, word of mouth, to physical flyers. With this, TapIn seems to clearly target a defined problem. Using this information, we further developed our articles and news sections to be personalized based on user selected interests. This feature was responded to positively in a few interviews. The games were also cited to be a strong feature of the app — and throughout the week, metrics seem to back the feedback up. As mentioned above, DailyFive and Pipes stand out specifically as low-friction, high-return features. Using this feedback, we pushed our effort towards fleshing out DailyFive and Pipes with intentional upgrades such as introducing a cloud-based leaderboard and AI-generated boards with increasing difficulty for Pipes, and push notifications for new DailyFive words. Social features were also a consistent trend across the interviews conducted. Some users mentioned that the app would benefit from a social layer on the article and events side — not just the games. We've begun iterating on this by integrating likes and comments, but have more to add on this end.
 
### Remaining Friction: The Events Page
 
The number one remaining hesitation seems to be the events page. Although cited as a main "promise" by many interviewees, it also seems to be the greatest point of friction. Many users mentioned that the events page UI was intimidating, overloaded with events, and lacking in information within individual events. Additionally, community engagement and calendar integration were cited as specific wants as opposed to just saving an event within the app. Now that we've identified that a) TapIn solves a problem and b) has a real pull, we need to focus our efforts on optimizing this magic part of our app.
 
### Features Planned for Final
 
- Flesh out friends/social layer — see who's interested and saved an event
    - Fix likes and comments
- Calendar integration — export events to personal calendar
- Event filtering & better event detail — filter by type, host photos/visuals, clarity on openness (if possible)
- Community features — comments, discussions, and voting on news and events
- Onboarding tutorial — brief intro to the app's top 3 features on first launch
- More notification controls — customizable alerts for saved events and daily game challenges
 
### UX Issues to Address
 
- Search bar exit flow is unintuitive — multiple users struggled to leave the search menu
- Bottom bar doesn't rise above keyboard in search
- "For You" label looks like a tappable button, not a section header
- Tap targets extend below the bottom bar, causing accidental event opens
- Red color on event dates/times reads as a warning — users suggested gold or white instead
- Share links direct to AggieLife (poor mobile experience) instead of back into TapIn
- AI event summaries sometimes reference features (e.g., a forum) that users can't actually access
- Echo is perceived as too difficult — multiple users quit immediately
