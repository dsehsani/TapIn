# Context
I am building an iOS app for UC Davis students using SwiftUI and MVVM architecture.
Here is the current state of the Events feature (Campus Tab):
- **Data Source:** Fetches live iCal data from `aggielife.ucdavis.edu`, parsed via a custom `ICalParser`.
- **Architecture:** MVVM (`CampusViewModel`, `CampusEvent` model).
- **UI:** A `CampusView` displays events in a `LazyVStack` of cards.
- **Current Card:** Shows Title, Date, Location, and a 2-line description preview.
- **Current Detail:** Clicking a card opens `EventDetailView` with the full description.

# The Task
I want to implement an **AI Summary Feature** for the Event Cards.

## Functional Requirements
1.  **Replacement Logic:** On the `CampusView` card, if an event has a **long description**, strictly replace the current 2-line preview text with a generated "AI Summary."
2.  **Conditional Trigger:** Only apply this AI summary if the original description text is long (e.g., > 100 characters). Short descriptions should display as they currently do.
3.  **Instant Loading:** There should be no "loading" or "thinking" animation. The summary must appear instantly when the card loads.
4.  **Source of Truth:** The summary is strictly derived from the `description` field parsed from the `.ical` file.
5.  **Detail View Unchanged:** When the user clicks the card, the `EventDetailView` must still show the **original full description**, not the summary.
6.  **Mocking Strategy:** Since I do not have the Claude API key yet, you must implement a logic that generates a fake "AI Summary" string for now so I can build the UI.

## Design Requirements (The "Gemini/Apple" Aesthetic)
The visual presentation of this summary on the card is crucial.
- **Style:** It needs to look like it was generated by modern AI (similar to the Apple Intelligence glow or the Google Gemini colorful ring).
- **Container:** The summary text should be contained in a pill, a bubble, or a specific section of the card that distinguishes it from the rest of the metadata.
- **Border/Glow:** Use a multi-color gradient (Blue, Purple, Gold, Pink) to create a "Rainbow Ring" or subtle glowing border effect around the summary text to indicate it is AI-generated.
- **Font:** Use a clean, perhaps slightly different font weight or style (like `rounded` design) for the summary text.

## Technical Implementation Steps
1.  Update the `CampusEvent` model to include logic that checks the description length.
2.  Create a visual ViewModifier or a custom View component named `AISummaryBadge` (or similar) that implements the rainbow/gradient style.
3.  Integrate this into the existing Event Card view, replacing the old text view conditionally.
