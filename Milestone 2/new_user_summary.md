## TapIn - Milestone 2 - Summary of Interviews:

App is primarily perceived as a news app (often first impression).
Events are important, but not always prioritized in navigation.

Brief confusion between Saved News vs. Saved Events.
Confusion around Save vs. Attend (does saving RSVP? does it notify?).

AI summaries are:
Helpful and compact.
Trusted moderately to highly.
Mostly scanned only in the first 1–2 lines.
Sometimes ignored or “tuned out.”
Sometimes seen as redundant with top stories.
Users want personalization (For You page, interest-based news/events).
Users like that AI is clearly labeled as AI.
Users like the visual AI design (glow, Gemini-like aesthetic).
Some users want the ability to toggle AI off.
Too much information on the main page for some users.

Events should include:
Reddit like popularity/friends going to make it more engaging.
Broader Davis community events (not just clubs). Even downtown events. 
Strong interest in:
Guest lectures.
Downtown events.
Community gatherings.
Desire for social features:
See how many people are attending.
See where friends are going.
University-wide leaderboard.

Games are:
Easy and intuitive.
Good for engagement.
Not seen as the core value of the app.
Streaks / points system would increase daily return.
Search bar not always noticeable.

Small UI issues:
Buttons too close together.
Leaderboard green squares unclear.
TBD location confusing.

App consistently described as:
Easy.
Intuitive.
Fast.
Clean UI.

Compared favorably to:
The Aggie.
University website.
Browsing on phone browser.

Security concerns raised around:
AI + user preferences.
Data safety.
